# SpaceWise

SpaceWise is a privacy-first Android storage explainer built with Kotlin, Jetpack Compose, and Material 3.

The project is intentionally conservative about what it claims. Android limits what ordinary apps can inspect, so SpaceWise distinguishes between:

- values measured directly from Android system APIs;
- values visible only after the user grants a permission;
- partial or unavailable categories;
- actions requested from Android but not yet verified as complete.

## Privacy model

- The manifest does not request `android.permission.INTERNET`.
- Storage analysis runs on the device through Android framework APIs.
- Cloud SDKs and network-client dependencies are not part of the application module.
- Android backup is disabled for application state.
- SpaceWise does not claim visibility into sandboxed system or app-private files.

This is a privacy-focused design, not an assertion that every Android build, dependency, or future contribution is automatically safe. Review the manifest and dependency graph before each release.

## Current measured features

- Total, used, and free space from `StatFs`.
- Media totals and large visible media items from `MediaStore`, subject to permission scope.
- Installed application storage statistics from `StorageStatsManager`, subject to Usage Access and package-visibility restrictions.
- Last-used application timestamps from `UsageStatsManager`, when available.
- Category residual storage shown as system/other rather than fabricated file listings.
- Deletion verification by re-querying `MediaStore` before reporting reclaimed bytes.

## Disabled prototype behavior

The earlier prototype included hard-coded Smart Clean entries, hard-coded large temporary files, synthetic thirty-day history, fixed savings estimates, and simulated reclaimed-storage offsets. Those values are not displayed as device findings in the hardened build.

Historical charts remain empty until real snapshots are persisted over time. Empty states are preferred over invented data.

## Permission behavior

| Capability | Access required | Without access |
|---|---|---|
| Total/free storage | None | Available |
| Photos | Image permission | Hidden or incomplete |
| Videos | Video permission | Hidden or incomplete |
| Audio | Audio permission | Hidden or incomplete |
| Selected Android 14 media | Partial visual access | Reported as partial, not full-device visibility |
| Application sizes and usage | Usage Access | Unavailable |
| Installed package inventory | Package visibility | Limited |
| Delete media | Android-mediated delete access | May fail or require confirmation |

SpaceWise currently uses `QUERY_ALL_PACKAGES` for internal/MVP package analysis. This permission requires a valid policy justification for Google Play distribution. A production Play Store build should narrow package visibility or use an approved declaration.

## Build requirements

- JDK 21
- Gradle 9.3.1
- Android SDK Platform 36
- Android Build Tools 36.0.0

Build and test:

```bash
gradle --no-daemon :app:testDebugUnitTest :app:assembleDebug
```

The repository CI workflow installs the matching JDK, Gradle, Android platform, and build tools, verifies the APK archive and signature, and uploads the debug APK as a workflow artifact.

## Release signing

A release build expects:

- `KEYSTORE_PATH`
- `STORE_PASSWORD`
- `KEY_ALIAS` (defaults to `upload`)
- `KEY_PASSWORD`

Do not commit a signing keystore or passwords. Debug APKs use the repository-local debug keystore generated by CI and are not production releases.

## Data truth rules

See [`docs/DATA_TRUTH_MODEL.md`](docs/DATA_TRUTH_MODEL.md). Every future storage feature should identify its source, permission requirements, confidence level, failure state, and verification method before it is exposed in the UI.

## Product direction

SpaceWise is a storage transparency tool first and a cleanup tool second. It should explain what Android can verify, what remains inaccessible, and what the user can safely review without imitating aggressive or misleading cleaner applications.
